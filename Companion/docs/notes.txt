iTerm2 Buddy (Companion App) Release Notes

Bug Fixes:
- Picking a session for a new session-bound chat
  now shows the same window > tab > pane list as
  the Sessions tab and the @-mention picker,
  instead of a flat, unorganized roster. Each row
  has an eye button to preview the session before
  committing to it, like the @-mention picker.
- Fast-scrolling a session's scrollback no longer
  floods the relay with tile requests, which
  could trip its rate limit and silently drop
  the connection to the Mac while the phone still
  looked connected. Requests are now paced.
- Reply and approval notifications are more
  reliable. A turn that pauses for your approval
  no longer leaves the typing indicator spinning,
  a command awaiting your Allow/Deny now shows as
  an action you need to take, and a chat you
  reconnect to mid-reply still notifies you when
  the reply arrives.

New Features:
- A resize button in the live session view fits
  the terminal grid to your phone for readable
  text: 40 columns in portrait, 80 in landscape,
  and as many rows as fill the screen. The Mac
  resizes the real session to match.
- You can now mute a chat so it stops sending
  push notifications to your phone. Swipe right
  on a chat in the list, or use the bell button
  in the conversation, to mute or unmute it. A
  muted chat shows a slashed bell in the list.
  The Mac remembers the choice, so pushes stay
  off even while your phone is unreachable.
  Requires a Mac new enough to support it; the
  controls are hidden otherwise.
- Message the agent from the live session view.
  A compose button opens an overlay with a text
  field and a voice-dictation mic, so you can send
  a message without leaving the session. If you
  reached the view by tapping an @-mention in a
  chat, your message goes to that chat; otherwise
  it goes to a session-bound chat. When the agent
  finishes replying, a notification appears;
  tapping it opens the chat, and Back returns you
  to the session view.
- Before your first send from a live session,
  iTerm2 Buddy can ask “Share This Session with
  the AI?” so the terminal's state and visible
  screen are included with your messages only
  after you allow it.
- Live session view. When the paired Mac is new
  enough, tapping a session shows live video that
  updates as the terminal changes, instead of a
  static snapshot. The Mac renders and streams the
  visible screen, so inline images, colors, and
  full-screen apps look exactly as on the Mac. A
  quiet terminal uses almost no data, the view
  stays current on a slow connection instead of
  lagging behind, and it pauses itself with a
  notice if it would use too much data.
  Scroll up to browse the session's scrollback,
  pinch to zoom in, and drag with one finger to
  pan around. Press and hold to start selecting
  text; drag the handles (with a magnifier) to
  adjust the selection, then Copy, Select All, or
  Paste from the menu. Selection works both in the
  live screen and in scrollback, and a selection
  made on the Mac shows on the phone too. The view
  follows along when you resize the Mac window.
  Older Macs keep showing the static snapshot.
- iTerm2 Buddy now asks for notification
  permission when you turn on sending terminal
  alerts to your iPhone on the Mac, instead of
  only when an AI chat needs them.
- The app now rotates to landscape on iPhone,
  not just portrait. (iPad already supported
  all orientations.)
- Messages from a newer iTerm2 that this app
  doesn't understand now show a placeholder
  ("You need a newer version of iTerm2 Buddy to
  view this message") instead of silently
  disappearing or hiding the rest of the chat.
- When you @-mention a session or workgroup in
  an orchestration chat, the agent is granted
  standing permission to control it without a
  prompt. A notice with a Revoke button lets
  you take that permission back.
- Voice dictation in chat. Tap the microphone
  in the message bar to dictate instead of type.
  The first time, it offers to download a small
  speech model; after that, tapping the mic
  starts listening right away.
  Transcription runs entirely on your device
  using Whisper, with no cloud and no cost. Your
  audio never leaves the phone.
  While listening you get a level meter and
  haptic feedback, the text appears as you speak,
  and tapping send (or leaving the chat) stops
  recording.
  Choose or remove the speech model under
  Settings > Voice Input. Only models that run
  well on this device are listed.
- The session picker for @-mentions now shows
  the same window, tab, and pane hierarchy as
  the Sessions tab. Each row has a preview
  button to view the session before picking it;
  tapping elsewhere on the row selects it.
- New chats now default to Orchestration on the
  New Chat screen.

Bug Fixes:
- Dictation no longer occasionally leaves the
  microphone recording, or wedges the mic button,
  when quickly switching between composer bars
  while starting or finishing a dictation.
- Double-tapping Send while dictating no longer
  shows a false "dictation was interrupted"
  message after the message was actually sent.
- A reply notification no longer occasionally
  drops a leading character (such as a minus
  sign) from the continued part of a long reply.
- A message sent from a session view that fails
  to send no longer leaves a duplicate, never-
  delivered bubble behind, even if you switch
  chats while it is sending. It comes back to
  the composer so you can retry.
- Reply notifications on the lock screen now
  always show session and workgroup names, even
  when the name appears late in a long reply.
- After a reconnect, an open chat no longer
  briefly reverts to an older set of messages,
  dropping ones that arrived since you last
  left it.
- Reply notifications for a session no longer
  keep firing after you leave it in the case
  where the same session was open in two
  places at once.
- The connection now recovers from brief network
  drops (like switching Wi-Fi or a spotty signal)
  instead of getting stuck on "Reconnecting." The
  Mac no longer becomes unreachable after a blip
  until it is relaunched.
- @-mentions in command-request messages now
  show the friendly session or workgroup name
  instead of the raw identifier.
- @-mentions in chat messages that could not be
  resolved (for example while the Mac was briefly
  unreachable) now show the session name once the
  connection recovers or you reopen the chat,
  instead of staying a raw identifier.
- In the Sessions tab, a split pane that hosts a
  peer workgroup now shows as a container with
  its own icon and is no longer tappable. Only
  the individual peers beneath it open a session.
- The relay servers moved to new addresses.
  Pairing now requires an up-to-date iTerm2
  and iTerm2 Buddy; a version that predates
  the move shows an update prompt instead of
  pairing into a state that cannot receive
  notifications.
- When the relay hits its daily data limit,
  the app now shows a “Daily data limit
  reached” banner with a Reconnect now
  button and stops retrying every couple
  seconds until the limit resets.
